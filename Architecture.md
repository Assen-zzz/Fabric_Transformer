# 光谱-视觉交叉注意力Transformer —— 模型架构与数据计算说明

## 一、任务概述

**目标**：对纺织品布料材质进行分类识别。

**输入**：
- **光谱特征**：近红外光谱 900-1700nm，1600维光谱数据，重构为 `40×40` 灰度图
- **图像特征**：纺织品布料材质的 `224×224` RGB视觉图像（CMOS工业相机采集）

**输出**：3类布料材质的分类结果

**核心思想**：光谱特征经EfficientNet提取后生成 **Q（查询）**，图像特征经ResNet-18改进型YOLO骨干+SPP提取后生成 **K/V（键/值）**，通过单向交叉注意力实现模态融合。

### 分步简化描述

| 步骤 | 模块 | 输入 → 输出 | 一句话说明 |
|------|------|------------|-----------|
| ① | 光谱EfficientNet | (B,1,40×40) → **(B,80,5,5)** | 光谱40×40图经4阶段MBConv提取到5×5特征图 |
| ② | 光谱通道映射 | (B,80,5,5) → **(B,25,256)** | 1×1卷积升维到256，reshape为25个token |

| ⑤ | 全局平均池化 | (B,25,256) → **(B,256)** | 25个token取平均，合并为1个全局特征 |
| ⑥ | 分类MLP | (B,256) → (B,128) → **(B,3)** | 两层全连接：256→128→3类 |

## 三、各阶段详细数据流与参数计算

### 3.1 第一阶段：光谱特征提取（EfficientNet微型版）

将1600维光谱重构为40×40灰度图，经EfficientNet微型版提取特征。

#### 3.1.1 光谱输入层

| 属性 | 值 |
|------|-----|
| 原始光谱维度 | 1600维 |
| 重构尺寸 | 40×40 |
| 输入张量形状 | (B, 1, 40, 40) |
| 数据来源 | 近红外光谱仪 900-1700nm |

#### 3.1.2 EfficientNet-Micro 逐层计算

```
输入: (B, 1, 40, 40)
  │
  │ Stem: Conv2d(1→32, 3×3, stride=1, padding=1) + BN + SiLU
  │ (40+2×1-3)/1+1=40 → 40×40
  │
  ▼
(B, 32, 40, 40)                                             参数: (1×32×3×3)+32 = 320
  │
  │ Stage1: MBConv1(3×3, stride=1, expand_ratio=1)
  │         深度可分离卷积, 通道32→16, 尺寸不变 40×40
  │
  ▼
(B, 16, 40, 40)                                             参数: ≈1,136
  │
  │ Stage2: MBConv6(3×3, stride=2, expand_ratio=6)
  │         先升维16→96, 深度卷积3×3, 降维96→24
  │         40×40 → 20×20 (stride=2)
  │
  ▼
(B, 24, 20, 20)                                             参数: ≈8,000
  │
  │ Stage3: MBConv6(5×5, stride=2, expand_ratio=6)
  │         先升维24→144, 深度卷积5×5, 降维144→40
  │         20×20 → 10×10 (stride=2)
  │
  ▼
(B, 40, 10, 10)                                             参数: ≈30,000
  │
  │ Stage4: MBConv6(3×3, stride=2, expand_ratio=6)
  │         先升维40→240, 深度卷积3×3, 降维240→80
  │         10×10 → 5×5 (stride=2)
  │
  ▼
(B, 80, 5, 5)                                               参数: ≈360,000
```

**参数总计**：EfficientNet-Micro ≈ **398,000 参数**

**各阶段特征图尺寸变化**：
```
40×40 ─→ 40×40 ─→ 20×20 ─→ 10×10 ─→ 5×5
(stage1)  (stage2)  (stage3)  (stage4)
```

#### 3.1.3 通道映射（1×1卷积）

将EfficientNet输出的80通道映射到d_model=256：

```
输入: (B, 80, 5, 5)
  │
  │ Conv2d(80→256, 1×1) + BN
  │ 1×1卷积保持空间尺寸5×5，通道从80升到256
  │
  ▼
输出: (B, 256, 5, 5)                                         参数: 80×256 + 256 = 20,736
  │
  │ reshape(B, 256, 25) → transpose → (B, 25, 256)
  │          ↑ 5×5=25个patches/tokens
  │
  ▼
最终: (B, 25, 256)   ← 25个光谱token，每个256维            ← Q
```

**token化过程**：
```
5×5特征图 → 展平 → 25个像素位置 → 每个位置256维特征 → 25个token
```

**关键点**：25个token对应光谱图上的25个空间位置，每个token代表该区域的局部光谱特征。

### 3.2 第二阶段：视觉特征提取（ResNet-18 + SPP）

#### 3.2.1 视觉输入层

| 属性 | 值 |
|------|-----|
| 输入张量形状 | (B, 3, 224, 224) |
| 色彩模式 | RGB |
| 数据来源 | CMOS工业相机 |

#### 3.2.2 ResNet-18 逐层计算（到layer3为止）

```
输入: (B, 3, 224, 224)
  │
  │ Conv1: Conv2d(3→64, 7×7, stride=2, padding=3)
  │ (224+6-7)/2+1=112 → 112×112
  │ + BN + ReLU
  │
  ▼
(B, 64, 112, 112)                                             参数: 3×64×7×7+64 = 9,472
  │
  │ MaxPool: MaxPool2d(3×3, stride=2, padding=1)
  │ (112+2-3)/2+1=56 → 56×56
  │
  ▼
(B, 64, 56, 56)                                               (无参数)
  │
  │ Layer1: ResBlock×2 (64→64, stride=1)
  │ 每个ResBlock: Conv3×3→BN→ReLU→Conv3×3→BN+残差
  │ 56×56 → 56×56 (stride=1)
  │
  ▼
(B, 64, 56, 56)                                               参数: 2×(64×64×3×3×2+64×2) = 73,984
  │
  │ Layer2: ResBlock×2 (64→128, stride=2)
  │ 第一个ResBlock stride=2: 56×56 → 28×28
  │ 通道从64升到128
  │
  ▼
(B, 128, 28, 28)                                              参数: 2×(64×128×3×3+128×128×3×3+128×2) ≈ 295,168
  │
  │ Layer3: ResBlock×2 (128→256, stride=2)
  │ 第一个ResBlock stride=2: 28×28 → 14×14
  │ 通道从128升到256
  │
  ▼
(B, 256, 14, 14)                                              参数: 2×(128×256×3×3+256×256×3×3+256×2) ≈ 1,180,672
```

**特征图下采样路径**：
```
224×224 ─→ 112×112 ─→ 56×56 ─→ 28×28 ─→ 14×14
(Conv1)   (MaxPool)  (Layer2)  (Layer3)
         ↓ 2        ↓ 2       ↓ 2
      224/2=112  112/2=56  56/2=28  28/2=14 → 最终14×14
```

**参数总计**：ResNet-18(→layer3) ≈ **1,559,296 参数**

#### 3.2.3 SPP模块（YOLO风格多尺度池化）

```
输入: (B, 256, 14, 14)
  │
  │ 三个并行池化分支:
  │   ├── MaxPool2d(5×5, padding=2) → (B,256,14,14)
  │   ├── MaxPool2d(9×9, padding=4) → (B,256,14,14)
  │   └── MaxPool2d(13×13, padding=6) → (B,256,14,14)
  │
  │ 拼接: concat(dim=1) → (B, 256×4, 14, 14) = (B, 1024, 14, 14)
  │
  │ Conv2d(1024→256, 1×1) 降维 + BN + SiLU
  │
  ▼
输出: (B, 256, 14, 14)                                         参数: 1,024×256+256 = 262,400
```

**SPP的作用**：
- 5×5池化：捕获局部(5/14≈35%感受野)纹理细节
- 9×9池化：捕获中等范围编织纹理
- 13×13池化：捕获全局布料褶皱/整体图案
- 原始特征保留局部细节

**参数总计**：SPP模块 ≈ **262,400 参数**

#### 3.2.4 Token化

```
(B, 256, 14, 14)
  │
  │ reshape(B, 256, 196) → transpose → (B, 196, 256)
  │          ↑ 14×14=196个patches/tokens
  │
  ▼
(B, 196, 256)   ← 196个图像token，每个256维            ← K, V
```

**关键点**：196个token对应图像上14×14网格，每个token代表图像上16×16像素区域的视觉特征（因为224/14=16）。

### 3.3 第三阶段：单向交叉注意力（核心模块）

#### 3.3.1 输入汇总

| 张量 | 来源 | 形状 | 含义 |
|------|------|------|------|
| **Q** | 光谱EfficientNet输出 | (B, 25, 256) | 25个光谱token，每个256维 |
| **K** | 视觉ResNet-18+SPP输出 | (B, 196, 256) | 196个图像token，每个256维 |
| **V** | 视觉ResNet-18+SPP输出 | (B, 196, 256) | 196个图像token，每个256维 |

**特点**：Q序列长度25 ≠ K/V序列长度196，这正是交叉注意力的优势所在。

#### 3.3.2 什么是交叉注意力？

交叉注意力（Cross-Attention）中 **Q来自一个模态（光谱），K和V来自另一个模态（图像）**，光谱token主动查询图像token中与自身最相关的区域。

**与自注意力的区别**：

| 类型 | Q来源 | K/V来源 | 作用 |
|------|-------|---------|------|
| 自注意力 | 自身 | 自身 | 同一模态内部建立关系 |
| 交叉注意力 | 光谱(25token) | 图像(196token) | 不同模态之间建立关系 |

#### 3.3.3 为什么Q(25)和K/V(196)的序列长度可以不同？

注意力核心公式：

```
Attention(Q, K, V) = softmax(Q × K^T / √d) × V
```

- Q: (B, 25, 256)
- K^T: (B, 256, 196)
- Q×K^T: (B, 25, 196) ← 矩阵乘法合法，256维匹配
- 输出: (B, 25, 256) ← **保留了Q的序列长度25**

**结论**：只要特征维度(256)一致，Q和K/V的序列长度可以任意不同。

#### 3.3.4 六步详细计算流程

##### Step 1：Q、K、V线性投影

```
Q_proj = Q × W_Q + b_Q      (B,25,256) → (B,25,256)
K_proj = K × W_K + b_K      (B,196,256) → (B,196,256)
V_proj = V × W_V + b_V      (B,196,256) → (B,196,256)
```

**参数**：每个Linear(256→256)有 256×256+256=65,792，三个共 197,376

##### Step 2：多头拆分（n_heads=4，每头64维）

```
Q_proj: (B, 25, 256) → reshape → (B, 25, 4, 64) → transpose → (B, 4, 25, 64)
K_proj: (B, 196, 256) → reshape → (B, 196, 4, 64) → transpose → (B, 4, 196, 64)
V_proj: (B, 196, 256) → reshape → (B, 196, 4, 64) → transpose → (B, 4, 196, 64)
```

##### Step 3：每头独立计算注意力分数

```
以头1为例：
Q_1: (B, 25, 64)  ← 光谱token的子空间特征
K_1: (B, 196, 64) ← 图像token的子空间特征
score_1 = Q_1 × K_1^T = (B, 25, 196)  ← 每个光谱token与所有图像token的相似度

4个头的score拼接: (B, 4, 25, 196)
```

##### Step 4：缩放 + Softmax

```
attn_weights = softmax(score / √64, dim=-1)
             = softmax(score / 8)

形状: (B, 4, 25, 196)
```

缩放原因：d_head=64时，点积方差≈64，除以√64=8防止softmax梯度饱和。

**Softmax函数定义**：

```
                    exp(zᵢ)                     eᶻⁱ
softmax(zᵢ) = ────────────────  =  ─────────────────
               Σⱼ exp(zⱼ)          eᶻ¹ + eᶻ² + ... + eᶻⁿ
```

##### Step 5：加权聚合V

```
output_h = attn_weights_h × V_h

头1: (B, 25, 196) × (B, 196, 64) → (B, 25, 64)
4头拼接: (B, 25, 256)
```

**物理含义**：每个光谱token的输出 = 所有196个图像token的加权求和，权重就是注意力分数。分数越高表示该图像patch与该光谱token越相关。

##### Step 6：输出投影 + 残差 + LayerNorm

```
output = Linear(256→256)(concat_heads) + Q_proj  ← 残差连接
output = LayerNorm(output)
output = Dropout(output, p=0.1)

最终形状: (B, 25, 256)
```

---

### 3.4 第四阶段：全局平均池化

```
输入: (B, 25, 256)
操作: 对25个token维度取均值
输出: (B, 256)
```

**无参数**，固定操作。将25个光谱token的融合特征合并为一个256维全局特征向量。

---

### 3.5 第五阶段：两层全连接分类头（MLP Head）

```
FC1: Linear(256 → 128) + GELU + Dropout(0.1)
     (B, 256) → (B, 128)
     参数: 256×128 + 128 = 32,896

FC2: Linear(128 → 3)
     (B, 128) → (B, 3)
     参数: 128×3 + 3 = 387
```

**GELU** 是比ReLU更平滑的激活函数：
```
GELU(x) = x × Φ(x) = 0.5 × x × (1 + tanh(√(2/π) × (x + 0.044715 × x³)))
```

## 六、多头注意力详解

### 6.1 什么是多头注意力？

多头注意力（Multi-Head Attention）将256维特征拆分为4个独立的64维子空间，每个头在不同子空间中独立计算注意力，最后拼接。

### 6.2 本模型中4个头的物理含义

| 头 | 关注维度 | 具体作用 |
|----|---------|---------|
| **头1** | 纹理相关性 | 光谱的编织纹理特征与图像中的编织图案是否匹配 |
| **头2** | 颜色相关性 | 光谱的吸收谷特征与图像中的颜色分布是否匹配 |
| **头3** | 反射/光泽 | 光谱的反射峰与图像中的高光/光泽区域是否匹配 |
| **头4** | 材质粗糙度 | 光谱的整体形状与图像中的材质粗糙/光滑程度是否匹配 |

### 6.3 为什么选择4头？头数取决于什么？

| 因素 | 2头 | 4头（选择） | 8头 |
|------|-----|-----------|-----|
| d_head | 128 | **64** | 32 |
| 多样性 | 低（仅2种视角） | **适中** | 高 |
| 每个头表达能力 | 强 | **适中** | 弱 |
| 计算量 | 少 | **适中** | 较多 |

**选择原因**：d_head=64是经过大量实践证明效果最好的子空间维度（BERT/GPT-2/ViT均使用d_head=64）。

## 七、模型特点总结

1. **双骨干网络**：光谱用EfficientNet-Micro（MBConv深度可分离卷积，高效轻量），视觉用ResNet-18+SPP（多尺度池化捕获全局上下文）
2. **单向交叉注意力融合**：光谱(25token)作为Query主动查询图像(196token)中的相关区域，替代传统简单拼接
3. **序列长度灵活**：Q(25)和K/V(196)长度不同，交叉注意力天然支持
4. **多头并行**：4个头在64维子空间中独立计算，捕获多维度特征关联
5. **中等规模**：约1220万参数，可在GPU上高效训练和推理
