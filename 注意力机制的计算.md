# 注意力机制的计算 —— Q、K、V 三者的完整交互过程

## 一、Q、K、V 各自的来源

在我们这个模型中，Q、K、V 来自**两个不同的模态**：

| 张量 | 来源 | 原始数据 | 经过的处理 |
|------|------|---------|-----------|
| **Q（Query，查询）** | 光谱分支 | 光谱灰度图 (B,1,16,16) | PatchEmbed → 光谱tokens → Q_linear |
| **K（Key，键）** | 视觉分支 | RGB图像 (B,3,224,224) | PatchEmbed → 图像tokens → K_linear |
| **V（Value，值）** | 视觉分支 | RGB图像 (B,3,224,224) | PatchEmbed → 图像tokens → V_linear |

**核心思想**：光谱的每个token作为"查询"，去图像的所有token中寻找相关信息。

```
光谱token #0: "我在光谱上看到这里有一块编织纹理，图像中哪里也有类似纹理？"
光谱token #1: "我在光谱上看到这里有一个吸收谷，图像中哪里颜色最深？"
...
```

---

## 二、进入注意力之前的形状

经过各自分支的处理后，送入注意力的三个张量形状为：

```
Q 形状: (B, 16, 256)    ← 16个光谱token，每个256维
K 形状: (B, 196, 256)   ← 196个图像token，每个256维
V 形状: (B, 196, 256)   ← 196个图像token，每个256维
```

**注意：Q的序列长度（16）≠ K/V的序列长度（196），但特征维度（256）必须相等。**

---

## 三、Step 1：多头拆分（reshape + transpose）

这是注意力计算的**第一步**，将256维特征均分为4个64维子空间。

### 3.1 为什么需要这一步？

如果不拆分，Q与K做点积：

```
Q (B, 16, 256) × K^T (B, 256, 196) → score (B, 16, 196)
```

结果是每个Q与所有K的一个相似度分数，**只有一个分数**。这意味着256维特征中所有的"纹理信息"、"颜色信息"、"形状信息"都混在一起计算一个分数。

**多头拆分的目的**：让模型在4个不同的64维子空间中**独立**计算4个不同的相似度分数，每个头专注于不同类型的特征。

### 3.2 reshape：切分维度

将最后1维256拆成 (4, 64)：

```
Q: (B, 16, 256) → reshape → (B, 16, 4, 64)
K: (B, 196, 256) → reshape → (B, 196, 4, 64)
V: (B, 196, 256) → reshape → (B, 196, 4, 64)
```

数字排列变化（只看Q的token #0）：
```
reshape前:
token #0: [a0, a1, a2, ..., a63, a64, ..., a127, a128, ..., a191, a192, ..., a255]
           └──────────┬──────────┘ └──────────┬──────────┘ └──────────┬──────────┘ └──────────┬──────────┘
                   头1的部分               头2的部分               头3的部分               头4的部分

reshape后:
token #0 → (4, 64)
  head_0: [a0, a1, ..., a63]       64维
  head_1: [a64, a65, ..., a127]    64维
  head_2: [a128, a129, ..., a191]  64维
  head_3: [a192, a193, ..., a255]  64维
```

### 3.3 transpose：重新排列维度

将"头"维度提到"token"维度前面，让每个头能访问所有token：

```
Q: (B, 16, 4, 64) → transpose(1, 2) → (B, 4, 16, 64)
K: (B, 196, 4, 64) → transpose(1, 2) → (B, 4, 196, 64)
V: (B, 196, 4, 64) → transpose(1, 2) → (B, 4, 196, 64)
```

**排列变化**：
```
transpose前（按token排列）：
batch token head dim
  1     0    0    [a0..a63]      ← token0的头1
  1     0    1    [a64..a127]    ← token0的头2
  1     0    2    [a128..a191]   ← token0的头3
  1     0    3    [a192..a255]   ← token0的头4
  1     1    0    [b0..b63]      ← token1的头1
  ...

transpose后（按头排列）：
batch head token dim
  1    0     0    [a0..a63]      ← 头1的token0
  1    0     1    [b0..b63]      ← 头1的token1
  ...共16个token...
  1    0     15   [p0..p63]      ← 头1的token15
  ───────────────────────────────────────
  1    1     0    [a64..a127]    ← 头2的token0
  ...
```

### 3.4 拆分后的结果

```
头1: Q₁ (B, 16, 64)    K₁ (B, 196, 64)    V₁ (B, 196, 64)
头2: Q₂ (B, 16, 64)    K₂ (B, 196, 64)    V₂ (B, 196, 64)
头3: Q₃ (B, 16, 64)    K₃ (B, 196, 64)    V₃ (B, 196, 64)
头4: Q₄ (B, 16, 64)    K₄ (B, 196, 64)    V₄ (B, 196, 64)
```

**四个头完全独立，形状相同，但内容不同**（来自原始256维的不同切片）。

### 3.5 为什么选择4个头？头的个数取决于什么？

#### 3.5.1 头数的硬约束：d_model 必须能被 n_heads 整除

这是最底层的数学约束。因为拆分操作是将 d_model 维均分为 n_heads 份：

```
d_head = d_model / n_heads

n_heads=2:  d_head = 256/2 = 128
n_heads=4:  d_head = 256/4 = 64      ← 我们选的
n_heads=8:  d_head = 256/8 = 32
n_heads=16: d_head = 256/16 = 16
```

**d_model 和 n_heads 必须满足 d_model % n_heads == 0**，否则无法均分。

#### 3.5.2 核心权衡：子空间维度 d_head 的选择

每个头的维度 d_head 是决定注意力效果的关键因素：

```
d_head 太小（如16） → 每个头表达能力不足，难以捕获复杂的特征相关性
d_head 适中（64）  → 经验上效果最好的区间
d_head 太大（128） → 头数太少，多样性不足，退化接近单头注意力
```

**经验规则**：d_head 通常在 **32~128** 之间。主流Transformer模型的选择：

| 模型 | d_model | n_heads | d_head | 
|------|---------|---------|--------|
| BERT-base | 768 | 12 | 64 |
| GPT-2 | 768 | 12 | 64 |
| ViT-base | 768 | 12 | 64 |
| **我们的模型** | **256** | **4** | **64** |
| GPT-3 | 12288 | 96 | 128 |

可以看到，d_head=64 是最主流的选择。

#### 3.5.3 在本模型中选择4头的具体原因

| 因素 | 2头 | 4头（选择） | 8头 |
|------|-----|-----------|-----|
| d_head | 128 | **64** | 32 |
| 注意力矩阵数 | 2个 | **4个** | 8个 |
| 每个头的表达能力 | 强 | **适中** | 弱 |
| 多样性 | 低（仅2种视角） | **适中** | 高 |
| 计算量 | 少 | **适中** | 较多 |

**选择4头的综合考量**：
- d_head=64 是经过大量实践证明效果最好的子空间维度
- 4个头的多样性足够覆盖"纹理、颜色、反光、形状"四种不同的特征维度
- 计算量适中，注意力分数矩阵大小为 (B, 4, 16, 196)
- 如果数据量增大或任务更复杂，可以升级到8头（调整d_model=512即可）

#### 3.5.4 调整头数的准则

```
增加头数的情况：                 减少头数的情况：
- d_model 较大（如512以上）        - d_model 较小（如128以下）
- 任务复杂，需要细粒度关注         - 数据量少，防止过拟合
- 计算资源充足                    - 需要更快的推理速度
- 希望模型捕获更多样化的特征关系    - 任务相对简单

调整方法：在 config.py 中修改 n_heads 参数即可
```

## 四、Step 2：注意力分数计算（Q × K^T）

每个头独立计算。以头1为例：

### 4.1 矩阵乘法

```
Q₁ (B, 16, 64) × K₁^T (B, 64, 196) → score₁ (B, 16, 196)
```

**为什么K要转置？**

矩阵乘法的维度匹配规则：最后一个维度必须相等。

```
Q₁: (B, 16, 64)
         ↑
   这64维要和K的64维匹配

K₁: (B, 196, 64)
            ↑
   64维在最后，需要转置才能和Q的64维匹配

K₁^T: (B, 64, 196)
         ↑
   64维提到前面，可以乘了

Q₁ (B, 16, 64) × K₁^T (B, 64, 196) → score₁ (B, 16, 196)
```

### 4.2 物理含义

```
score₁[0][5] = Q₁[0] · K₁[5] = 光谱token #0 与 图像token #5 的点积相似度

score₁ 矩阵 (1, 16, 196)：

        图像token #0  #1  #2 ... #195
         ┌─────────────────────────┐
光谱    0│ 0.8  0.1  0.3 ...  0.2 │  ← token0与所有图像token的相似度
token   1│ 0.2  0.9  0.1 ...  0.1 │
        2│ 0.3  0.2  0.7 ...  0.4 │
      ...│ ...  ...  ... ...  ... │
       15│ 0.1  0.3  0.2 ...  0.9 │
         └─────────────────────────┘
```

**数值含义**：
- 0.8 表示"光谱token #0 与 图像token #0 非常相关"
- 0.1 表示"光谱token #0 与 图像token #1 几乎无关"
- 每行的值之和没有约束（还没做softmax）

### 4.3 四个头输出

```
头1: score₁ (B, 16, 196)  ← 关注"纹理相关性"
头2: score₂ (B, 16, 196)  ← 关注"颜色相关性"
头3: score₃ (B, 16, 196)  ← 关注"反光相关性"
头4: score₄ (B, 16, 196)  ← 关注"形状相关性"
```

## 五、Step 3：缩放 + Softmax

### 5.1 为什么要缩放？

当维度d_head=64时，点积的方差约为64，导致结果集中在0附近的大数值区域。softmax在输入很大时梯度极小（梯度饱和），不利于训练。

**解决方案**：将点积结果除以 √d_head = √64 = 8。

```
score_scaled = score / 8
```

### 5.2 Softmax 操作

#### Softmax 函数定义式

Softmax 是一种将任意实数向量转换为概率分布的函数。对于向量 **z** = [z₁, z₂, ..., zₙ]，softmax 定义为：

```
                    exp(zᵢ)                     eᶻⁱ
softmax(zᵢ) = ────────────────  =  ─────────────────
               Σⱼ exp(zⱼ)          eᶻ¹ + eᶻ² + ... + eᶻⁿ
```

其中：
- **zᵢ** 是输入向量的第 i 个元素（注意力分数）
- **exp(zᵢ) = eᶻⁱ** 是以自然常数 e 为底的指数函数
- **分子**：当前元素的指数值
- **分母**：所有元素指数值的总和（归一化因子）

**输出性质**：
- 每个输出值在 (0, 1) 之间
- 所有输出值之和等于 1
- 输入中较大的值会被"放大"（指数效应），较小的值被"压制"

**在我们的模型中**：

对每个光谱token（每一行）独立做softmax，将分数转为概率分布：

```
attn_weights₁ = softmax(score₁ / 8, dim=-1)

对每一行（每个光谱token）：
  ① 取指数: exp(score[j] / 8)
  ② 归一化: exp(score[j] / 8) / Σₖ exp(score[k] / 8)
```

用具体数字演示 softmax 的效果（简化到3个图像token）：

```
假设某光谱token的原始分数: [2.0, 1.0, 0.1]

第1步：取指数
  exp(2.0) = 7.39
  exp(1.0) = 2.72
  exp(0.1) = 1.11
  总和 = 7.39 + 2.72 + 1.11 = 11.22

第2步：归一化
  softmax(2.0) = 7.39 / 11.22 = 0.66  ← 最大（66%的注意力）
  softmax(1.0) = 2.72 / 11.22 = 0.24  ← 中等
  softmax(0.1) = 1.11 / 11.22 = 0.10  ← 最小

结果: [0.66, 0.24, 0.10]  ← 和=1.0，成为概率分布
```

**指数效应的关键作用**：2.0 和 1.0 的原始差距是2倍，但经过 softmax 后变成 0.66 vs 0.24 ≈ 2.75倍——**强者越强，弱者越弱**。这使得模型能更明确地区分"重要"和"不重要"的图像patch。

**结果**：

```
attn_weights₁ (1, 16, 196)：

        图像token #0   #1   #2  ... #195
         ┌──────────────────────────┐
光谱    0│ 0.15  0.02  0.08 ... 0.01│  ← 这一行求和=1.0
token   1│ 0.03  0.25  0.02 ... 0.02│
        2│ 0.06  0.03  0.18 ... 0.07│
      ...│ ...   ...   ... ...  ...│
       15│ 0.01  0.04  0.03 ... 0.20│
         └──────────────────────────┘
```

**注意**：现在**每一行的值加起来等于1**，表示该光谱token对196个图像token的注意力权重分布。

- 0.15 表示"光谱token #0 分配15%的注意力给图像token #0"
- 0.02 表示"光谱token #0 只分配2%的注意力给图像token #1"
- 这个分布决定了：光谱token #0 主要关注图像中的哪些区域

## 六、Step 4：加权聚合（score × V）

### 6.1 计算

```
out₁ = attn_weights₁ (B, 16, 196) × V₁ (B, 196, 64) → out₁ (B, 16, 64)
```

### 6.2 详细计算（B=1，单一token为例）

对光谱token #0：

```
out₁[0] = attn_weights₁[0][0] × V₁[0]   ← 图像token #0 的贡献
        + attn_weights₁[0][1] × V₁[1]   ← 图像token #1 的贡献
        + attn_weights₁[0][2] × V₁[2]   ← 图像token #2 的贡献
        + ...
        + attn_weights₁[0][195] × V₁[195] ← 图像token #195 的贡献
```

**物理含义**：光谱token #0 的64维输出，是**所有196个图像token的64维特征的加权平均**，权重就是softmax后的注意力分数。

### 6.3 一个具体的数值例子

假设简化到只有3个图像token，64维压缩到2维：

```
attn_weights[0] = [0.7, 0.2, 0.1]   ← 光谱token #0 关注图像token #0最多

V₁:
  token0: (0.8, 0.3)    ← 图像的纹理特征
  token1: (0.1, 0.9)    ← 图像的颜色特征
  token2: (0.5, 0.5)    ← 图像的混合特征

out₁[0] = 0.7 × (0.8, 0.3) + 0.2 × (0.1, 0.9) + 0.1 × (0.5, 0.5)
       = (0.56, 0.21) + (0.02, 0.18) + (0.05, 0.05)
       = (0.63, 0.44)
```

**最终输出**：(0.63, 0.44) 中，0.63接近token0的0.8（因为权重0.7最高），说明这个光谱token主要吸收了图像token0的特征。

### 6.4 四个头的输出

```
头1: out₁ (B, 16, 64)  ← 融合了"纹理维度"的图像信息
头2: out₂ (B, 16, 64)  ← 融合了"颜色维度"的图像信息
头3: out₃ (B, 16, 64)  ← 融合了"反光维度"的图像信息
头4: out₄ (B, 16, 64)  ← 融合了"形状维度"的图像信息
```

## 七、Step 5：多头合并

### 7.1 concat

将四个头的结果拼接回256维：

```
四个头: [(B,16,64), (B,16,64), (B,16,64), (B,16,64)]

concat: (B, 4, 16, 64) → transpose → (B, 16, 4, 64) → reshape → (B, 16, 256)
```

### 7.2 数据排列还原

```
transpose前（按头排列）：
batch head token dim
  1    0    0    [out₁_0..out₁_63]     ← 头1的输出
  1    1    0    [out₂_64..out₂_127]   ← 头2的输出
  ...

transpose+reshape后（按token排列）：
batch token dim
  1    0    [out₁_0..out₁_63 | out₂_64..out₂_127 | ... | out₄_192..out₄_255]
             └──────头1──────┘ └──────头2──────┘       └──────头4──────┘
```

**每个token的256维 = 4个头的64维拼接**，包含了四个不同子空间的注意力信息。

---

## 八、Step 6：输出投影 + 残差连接 + LayerNorm

### 8.1 输出投影

```
concat_out (B, 16, 256) → Linear(256→256) → (B, 16, 256)
```

这是对多头拼接结果的**一次线性变换**，让四个头的输出信息进一步融合。

### 8.2 残差连接

```
output = output_proj + Q_proj

其中 Q_proj 是 Q_linear 的输出（即最初的Q经过线性投影后的结果）
```

**残差连接的作用**：让梯度可以直接流回前面的层，缓解梯度消失，让模型更容易训练。

### 8.3 LayerNorm + Dropout

```
output = LayerNorm(output)
output = Dropout(output, p=0.1)
```

**最终输出**：(B, 16, 256)

## 十、关键维度变化总结

| 步骤 | 操作 | Q维度 | K维度 | V维度 | 输出维度 |
|------|------|-------|-------|-------|---------|
| 输入 | — | (B,16,256) | (B,196,256) | (B,196,256) | — |
| ① | 多头拆分 | (B,4,16,64) | (B,4,196,64) | (B,4,196,64) | — |
| ② | Q×K^T | — | — | — | (B,4,16,196) |
| ③ | Softmax | — | — | — | (B,4,16,196) |
| ④ | ×V | — | — | — | (B,4,16,64) |
| ⑤ | 多头合并 | — | — | — | (B,16,256) |
| ⑥ | 输出投影 | — | — | — | (B,16,256) |

**没有展平**：在整个注意力过程中，**始终保留着Q的序列维度（16）**，直到全局平均池化才变成向量。
