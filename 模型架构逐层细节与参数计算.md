# 模型架构逐层细节与参数计算

## 概述

本文档详细展示模型中 **每一层** 的输入/输出形状变化、卷积核参数、线性层权重维度、注意力计算细节，从原始输入到最终分类输出的完整计算链路。

---

## 一、光谱分支（生成Q）：从灰度图到Token序列

### 1.1 输入层

| 属性 | 值 |
|------|-----|
| 输入张量形状 | (B, 1, 16, 16) |
| 含义 | Batch × 通道(灰度) × 高 × 宽 |
| 数据来源 | 光谱曲线等距采样256点，reshape为16×16 |

### 1.2 PatchEmbed 层（一次卷积完成）

这是一次卷积完成分块映射，**不需要多次卷积+池化**。

| 属性 | 值 |
|------|-----|
| **卷积层** | Conv2d |
| 输入通道 | 1 |
| 输出通道 | 256 |
| 卷积核大小 | 4 × 4 |
| 步长 stride | 4 |
| 填充 padding | 0 |
| 卷积核数量 | **256个**（每个输出通道一个4×4核） |
| 每个卷积核参数 | 1 × 4 × 4 = 16个权重 + 1个偏置（共享） |
| 卷积核总参数量 | 256 × (1×4×4) = 4,096 权重 + 256 偏置 = **4,352** |

**计算过程**：

```
输入: (B, 1, 16, 16)
  │
  │ 用256个4×4卷积核，步长4扫过整个16×16图像
  │ 每次滑窗覆盖 4×4 区域，步长4意味着不重叠
  │ 输出尺寸 = (16 - 4)/4 + 1 = 4
  │
  ▼
卷积输出: (B, 256, 4, 4)
  │
  │ 将后两维展平: reshape(B, 256, 4×4) = (B, 256, 16)
  │ 转置最后两维: transpose → (B, 16, 256)
  │
  ▼
输出: (B, 16, 256)  ← 16个token，每个256维
```

**视觉理解**：
- 16×16 图像被 4×4 不重叠滑窗切成 (16/4)² = 4×4 = **16个patch**
- 每个patch（4×4=16个像素）被256个卷积核分别提取特征，得到256维向量
- 所以最终：16个patch × 256维特征 = **16个token**

### 1.3 LayerNorm

| 属性 | 值 |
|------|-----|
| 层 | LayerNorm |
| 归一化维度 | 256 |
| 参数 | 增益γ: 256个, 偏置β: 256个 = **512个参数** |

### 1.4 Q_linear 投影层

| 属性 | 值 |
|------|-----|
| 层 | Linear(in=256, out=256) |
| 权重矩阵维度 | W: (256, 256) |
| 偏置维度 | b: (256,) |
| 参数量 | 256 × 256 + 256 = **65,792** |

**计算过程**：

```
输入: (B, 16, 256)
  │
  │ Q_proj = X × W^T + b
  │ 每个token的256维向量经过线性变换，仍然是256维
  │ 这是对特征空间的重新映射，不改变形状
  │
  ▼
输出: (B, 16, 256)  ← 形状不变，特征空间被优化
```

## 二、视觉分支（生成K、V）：从RGB图到Token序列

### 2.1 输入层

| 属性 | 值 |
|------|-----|
| 输入张量形状 | (B, 3, 224, 224) |
| 含义 | Batch × RGB通道 × 高 × 宽 |

### 2.2 PatchEmbed 层

| 属性 | 值 |
|------|-----|
| **卷积层** | Conv2d |
| 输入通道 | 3（RGB） |
| 输出通道 | 256 |
| 卷积核大小 | 16 × 16 |
| 步长 stride | 16 |
| 填充 padding | 0 |
| 卷积核数量 | **256个**（每个输出通道一个16×16核） |
| 每个卷积核参数 | 3 × 16 × 16 = 768个权重 + 1个偏置 |
| 卷积核总参数量 | 256 × (3×16×16) = 196,608 权重 + 256 偏置 = **196,864** |

**计算过程**：

```
输入: (B, 3, 224, 224)
  │
  │ 用256个16×16卷积核，步长16扫过整个224×224图像
  │ 每次滑窗覆盖 16×16×3 区域，步长16意味着不重叠
  │ 输出尺寸 = (224 - 16)/16 + 1 = 14
  │
  ▼
卷积输出: (B, 256, 14, 14)
  │
  │ 将后两维展平: reshape(B, 256, 14×14) = (B, 256, 196)
  │ 转置最后两维: transpose → (B, 196, 256)
  │
  ▼
输出: (B, 196, 256)  ← 196个token，每个256维
```

**视觉理解**：
- 224×224 图像被 16×16 不重叠滑窗切成 (224/16)² = 14×14 = **196个patch**
- 每个patch（16×16×3=768个值）被256个卷积核提取特征，得到256维向量
- 所以最终：196个patch × 256维特征 = **196个token**

### 2.3 LayerNorm

| 属性 | 值 |
|------|-----|
| 层 | LayerNorm |
| 归一化维度 | 256 |
| 参数 | 256 × 2 = **512个参数** |

### 2.4 K_linear 和 V_linear 投影层

| 属性 | K_linear | V_linear |
|------|----------|----------|
| 层 | Linear(256,256) | Linear(256,256) |
| 权重维度 | (256, 256) | (256, 256) |
| 偏置维度 | (256,) | (256,) |
| 参数量 | 65,792 | 65,792 |

**计算过程**：

```
输入图像tokens: (B, 196, 256)
         │
    ┌────┴────┐
    ▼         ▼
K_proj      V_proj
(B,196,256) (B,196,256)
    │         │
    └────┬────┘
         ▼
   送入CrossAttention
```

## 三、CrossAttention（核心模块）

### 3.1 输入汇总

| 张量 | 来源 | 形状 |
|------|------|------|
| Q | 光谱分支经过Q_linear | (B, 16, 256) |
| K | 视觉分支经过K_linear | (B, 196, 256) |
| V | 视觉分支经过V_linear | (B, 196, 256) |

### 3.2 多头拆分（n_heads=4）

```
Q: (B, 16, 256) → reshape → (B, 16, 4, 64) → transpose → (B, 4, 16, 64)
K: (B, 196, 256) → reshape → (B, 196, 4, 64) → transpose → (B, 4, 196, 64)
V: (B, 196, 256) → reshape → (B, 196, 4, 64) → transpose → (B, 4, 196, 64)
```

**拆分逻辑**：
- 将256维特征均分为4份，每份64维
- 每个头在独立的64维子空间中计算注意力

### 3.3 注意力分数计算（每头独立）

```
头1: Q_1 (B, 16, 64) × K_1^T (B, 64, 196) → score_1 (B, 16, 196)
头2: Q_2 (B, 16, 64) × K_2^T (B, 64, 196) → score_2 (B, 16, 196)
头3: Q_3 (B, 16, 64) × K_3^T (B, 64, 196) → score_3 (B, 16, 196)
头4: Q_4 (B, 16, 64) × K_4^T (B, 64, 196) → score_4 (B, 16, 196)
```

**含义**：每个光谱token（共16个）计算与所有图像token（共196个）的相似度。

### 3.4 缩放 + Softmax

```
attn_weights = softmax(score / √64)
             = softmax(score / 8.0)

形状: (B, 4, 16, 196)
```

**缩放原因**：当维度较高时，点积结果可能很大，导致softmax梯度消失。除以√d_head=8使数值稳定。

### 3.5 加权聚合V

```
头1: attn_1 (B, 16, 196) × V_1 (B, 196, 64) → out_1 (B, 16, 64)
头2: attn_2 (B, 16, 196) × V_2 (B, 196, 64) → out_2 (B, 16, 64)
头3: attn_3 (B, 16, 196) × V_3 (B, 196, 64) → out_3 (B, 16, 64)
头4: attn_4 (B, 16, 196) × V_4 (B, 196, 64) → out_4 (B, 16, 64)
```

**含义**：用注意力权重对图像特征V进行加权求和，权重越高表示该图像patch与当前光谱token越相关。

### 3.6 多头合并

```
concat: (B, 4, 16, 64) → transpose → (B, 16, 4, 64) → reshape → (B, 16, 256)
```

### 3.7 输出投影 + 残差连接 + LayerNorm

| 操作 | 输入 | 输出 |
|------|------|------|
| Linear(256→256) | (B,16,256) | (B,16,256) |
| + Q_proj（残差） | — | (B,16,256) |
| LayerNorm | (B,16,256) | (B,16,256) |
| Dropout(0.1) | (B,16,256) | (B,16,256) |

**残差连接参数量**：Linear(256→256) = 65,792

**最终输出**：(B, 16, 256) —— 保留了光谱的序列长度，但每个token已融合了图像信息。

## 四、全局平均池化

| 属性 | 值 |
|------|-----|
| 操作 | torch.mean(dim=1) |
| 输入 | (B, 16, 256) |
| 输出 | (B, 256) |
| 参数量 | **0**（无参数，固定操作） |

**物理含义**：将16个光谱token的向量取平均值，合并为一个256维的全局特征向量。这个向量同时包含了光谱信息和通过注意力融合的图像信息。

## 五、分类MLP Head（两层全连接）

### 5.1 FC1: Linear(256 → 128)

| 属性 | 值 |
|------|-----|
| 层 | Linear(in=256, out=128) |
| 权重矩阵 | W: (128, 256) |
| 偏置 | b: (128,) |
| 参数量 | 256 × 128 + 128 = **32,896** |

**计算过程**：

```
输入: (B, 256)
  │
  │ out = X × W^T + b
  │ W形状 (128, 256)，每个输出神经元连接所有256个输入
  │
  ▼
输出: (B, 128)
  │
  │ GELU激活函数：非线性变换
  │ GELU(x) = x × Φ(x)，其中Φ是标准正态分布的CDF
  │
  ▼
  (B, 128)
  │
  │ Dropout(0.1)：随机丢弃10%的神经元，防止过拟合
  │
  ▼
输出: (B, 128)
```

GELU是比ReLU更平滑的激活函数，公式为 `GELU(x) = 0.5 × x × (1 + tanh(√(2/π) × (x + 0.044715 × x³)))`。

### 5.2 FC2: Linear(128 → 3)

| 属性 | 值 |
|------|-----|
| 层 | Linear(in=128, out=3) |
| 权重矩阵 | W: (3, 128) |
| 偏置 | b: (3,) |
| 参数量 | 128 × 3 + 3 = **387** |

**计算过程**：

```
输入: (B, 128)
  │
  │ out = X × W^T + b
  │ W形状 (3, 128)，3个输出对应3类布料材质
  │
  ▼
输出: (B, 3)  ← 3个logits，对应3类材质的原始分数
```

**最终输出解释**：
- 输出形状为 (B, 3)，每个样本得到3个实数（logits）
- 在训练时：logits 送入 CrossEntropyLoss 计算损失
- 在推理时：`torch.argmax(logits, dim=1)` 取最大值索引即为预测类别

## 七、卷积核可视化理解

### 光谱分支的卷积核

```
光谱输入: 16×16 灰度图

    4×4卷积核 #1 → 提取局部光谱纹理特征
    4×4卷积核 #2 → 提取局部梯度特征
    4×4卷积核 #3 → 提取局部均值特征
    ...
    4×4卷积核 #256 → 提取第256种局部特征

每个卷积核在16×16图上以步长4滑动，产生4×4=16个响应值
所有256个核的响应堆叠成: 256通道 × 4×4
```

### 图像分支的卷积核

```
图像输入: 224×224 RGB图

    16×16卷积核 #1 → 提取局部布料纹理（如编织图案）
    16×16卷积核 #2 → 提取局部颜色分布
    16×16卷积核 #3 → 提取边缘/轮廓
    ...
    16×16卷积核 #256 → 提取第256种局部特征

每个卷积核在224×224图上以步长16滑动，产生14×14=196个响应值
所有256个核的响应堆叠成: 256通道 × 14×14
```

**注意**：这里只有 **一次卷积**，没有多次卷积和池化堆叠。这是ViT（Vision Transformer）风格的PatchEmbed，与CNN风格的多次卷积+池化不同。

## 九、推理计算量估算（FLOPs）

| 模块 | 一次前向的乘加操作数（FLOPs） |
|------|----------------------------|
| 光谱Conv2d(1→256,4×4) | B × 256 × 4 × 4 × 16 = B × 65,536 |
| 图像Conv2d(3→256,16×16) | B × 256 × 16 × 16 × 3 × 196 = B × 38,535,168 |
| Q/K/V投影(各一次) | 3 × B × 16 × 256 × 256 = B × 3,145,728 |
| 注意力分数(4头) | B × 4 × 16 × 64 × 196 = B × 802,816 |
| 注意力加权(4头) | B × 4 × 16 × 64 × 196 = B × 802,816 |
| 输出投影 | B × 16 × 256 × 256 = B × 1,048,576 |
| FC1+FC2 | B × (256×128 + 128×3) = B × 33,152 |
| **总计（B=1）** | **≈ 44.4M FLOPs**（约4400万次乘加） |

这属于轻量级模型，在GPU上可以快速推理。
