# 光谱-图像单向交叉注意力Transformer —— 模型架构与数据计算说明

## 一、任务概述

**目标**：对布料材质进行分类识别。

**输入**：
- **光谱特征**：一条光谱曲线等距采样256个点，reshape为 `16×16` 灰度图
- **图像特征**：布料材质的 `224×224` RGB视觉图像

**输出**：3类布料材质的分类结果

**核心思想**：光谱特征生成 **Q（查询）**，图像特征生成 **K/V（键/值）**，通过单向交叉注意力实现模态融合。

| 步骤 | 模块 | 输入 → 输出 | 一句话说明 |
|------|------|------------|-----------|
| ① | 光谱PatchEmbed | (B,1,16×16) → **(B,16,256)** | 光谱图分为16个patch，每个映射到256维 |

## 三、各阶段详细数据流与参数计算

### 3.1 第一阶段：PatchEmbed（图像分块嵌入）

将图像分割为固定大小的patch，每个patch映射到d_model维向量。

#### 光谱分支（生成Q）

| 步骤 | 操作 | 输入形状 | 输出形状 | 参数说明 |
|------|------|---------|---------|---------|
| 1 | Conv2d(k=4,s=4,p=0) | (B,1,16,16) | (B,256,4,4) | 卷积核4×4，步长4，输出通道256 |
| 2 | Flatten H,W | (B,256,4,4) | (B,256,16) | 4×4=16个patch |
| 3 | Transpose | (B,256,16) | **(B,16,256)** | 变为token序列格式 |

**参数计算**：
```
Conv2d(1→256, 4×4) 权重: 1×256×4×4 = 4,096
偏置: 256
LayerNorm(256): 256×2 = 512
合计: 4,096 + 256 + 512 = 4,864
```

#### 图像分支（生成K、V）

| 步骤 | 操作 | 输入形状 | 输出形状 | 参数说明 |
|------|------|---------|---------|---------|
| 1 | Conv2d(k=16,s=16,p=0) | (B,3,224,224) | (B,256,14,14) | 卷积核16×16，步长16，输出通道256 |
| 2 | Flatten H,W | (B,256,14,14) | (B,256,196) | 14×14=196个patch |
| 3 | Transpose | (B,256,196) | **(B,196,256)** | 变为token序列格式 |

**参数计算**：
```
Conv2d(3→256, 16×16) 权重: 3×256×16×16 = 196,608
偏置: 256
LayerNorm(256): 256×2 = 512
合计: 196,608 + 256 + 512 = 197,376
```

---

### 3.2 第二阶段：单向交叉注意力（核心模块）

#### 3.2.1 什么是交叉注意力？

交叉注意力（Cross-Attention）是Transformer中的一种注意力机制，其中 **Q来自一个模态（光谱），K和V来自另一个模态（图像）**。这使得光谱特征能够"查询"图像特征中与自身相关的信息。

**与自注意力的区别**：

| 类型 | Q来源 | K/V来源 | 作用 |
|------|-------|---------|------|
| 自注意力 | 自身 | 自身 | 同一模态内部建立关系 |
| 交叉注意力 | 模态A | 模态B | 不同模态之间建立关系 |

#### 3.2.2 为什么Q的序列长度(16)和K/V的序列长度(196)可以不同？

注意力计算的核心公式：

```
Attention(Q, K, V) = softmax(Q × K^T / √d) × V
```

- Q形状: (B, 16, d_model)
- K^T形状: (B, d_model, 196)
- Q×K^T形状: (B, 16, 196) ← 矩阵乘法合法，因为Q的最后一维(d_model)与K^T的第一维(d_model)匹配
- 输出形状: (B, 16, d_model) ← 保留了Q的序列长度

**结论：Q和K/V的序列长度不需要相等，只需要特征维度(d_model)一致即可。**

#### 3.2.3 详细计算流程

##### Step 1：Q、K、V线性投影

```
Q_proj = Q × W_Q + b_Q      (B,16,256) → (B,16,256)
K_proj = K × W_K + b_K      (B,196,256) → (B,196,256)
V_proj = V × W_V + b_V      (B,196,256) → (B,196,256)
```

**参数**：每个Linear(256→256)有 256×256 + 256 = 65,792 个参数，三个共 197,376

##### Step 2：多头拆分（n_heads=4，每头d_head=64）

```
Q_proj: (B, 16, 256) → reshape → (B, 16, 4, 64) → transpose → (B, 4, 16, 64)
K_proj: (B, 196, 256) → reshape → (B, 196, 4, 64) → transpose → (B, 4, 196, 64)
V_proj: (B, 196, 256) → reshape → (B, 196, 4, 64) → transpose → (B, 4, 196, 64)
```

##### Step 3：每头独立计算注意力分数

```
score_h = Q_h × K_h^T

以头1为例：
Q_1: (B, 16, 64)
K_1^T: (B, 64, 196)
score_1: (B, 16, 196)    ← 每个光谱token与所有图像token的相似度

4个头拼接: (B, 4, 16, 196)
```

##### Step 4：缩放 + Softmax

```
attn_weights = softmax(score / √64)
             = softmax(score / 8)

形状: (B, 4, 16, 196)    ← 注意力权重，可解释性强
```

##### Step 5：加权聚合V

```
output_h = attn_weights_h × V_h

头1: (B, 16, 196) × (B, 196, 64) → (B, 16, 64)
4头拼接: (B, 16, 256)
```

##### Step 6：输出投影 + 残差 + LayerNorm

```
output = Linear(256→256)(concat_heads) + Q_proj  ← 残差连接
output = LayerNorm(output)
output = Dropout(output)

最终形状: (B, 16, 256)
```

---

### 3.3 第三阶段：全局平均池化

```
输入: (B, 16, 256)
操作: 对16个token维度取均值
输出: (B, 256)
```

**无参数**，固定操作。

### 3.4 第四阶段：两层全连接分类头（MLP Head）

```
FC1: Linear(256 → 128) + GELU + Dropout(0.1)
     (B, 256) → (B, 128)
     参数: 256×128 + 128 = 32,896

FC2: Linear(128 → 3)
     (B, 128) → (B, 3)
     参数: 128×3 + 3 = 387
```

## 五、多头注意力详解

### 5.1 什么是多头注意力？

多头注意力（Multi-Head Attention）将模型拆分为多个独立的"头"，每个头在不同的子空间中独立计算注意力，最后将各头的结果拼接起来。

### 5.2 数学定义

```
MultiHead(Q, K, V) = Concat(head_1, ..., head_h) × W_O

其中 head_i = Attention(Q × W_Q_i, K × W_K_i, V × W_V_i)
```

### 5.3 单头 vs 多头的对比

| 方面 | 单头注意力 | 多头注意力（4头） |
|------|-----------|-----------------|
| 子空间数量 | 1个（256维） | 4个（每头64维） |
| 注意力分布 | 每个Q产生1组权重 | 每个Q产生4组独立权重 |
| 特征捕获 | 单一视角 | 多视角并行 |
| 表达能力 | 有限 | 更强，可学习多种关系 |
| 可解释性 | 简单但信息少 | 每头可独立可视化分析 |

### 5.4 物理意义类比

将多头注意力想象成一组专家同时会诊：

- **头1**：关注"光谱的纹理特征"与"图像中的编织纹理"是否匹配
- **头2**：关注"光谱的吸收谷特征"与"图像中的颜色区域"是否匹配
- **头3**：关注"光谱的反射峰"与"图像中的高光区域"是否匹配
- **头4**：关注"光谱的整体形状"与"图像中的材质粗糙度"是否匹配

每个头独立做注意力计算，最后把4个专家的结论合并成一个综合判断。

## 六、模型特点总结

1. **单向融合**：光谱→图像，光谱作为Query主动查询图像中的相关区域
2. **序列长度灵活**：Q(16个token)和K/V(196个token)长度不同，交叉注意力天然支持
3. **多头并行**：4个头在64维子空间中独立计算，捕获多维度特征关联
4. **轻量级**：约49.9万参数，适合快速训练和部署
5. **可解释性强**：可提取注意力权重进行可视化，分析模型关注区域
